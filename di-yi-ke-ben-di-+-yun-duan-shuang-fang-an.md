### 一、 Python 运行机制与环境对比 (10 分钟)

#### 1. 解释型 vs 编译型

* 解释型 (Python): 源代码 $$ $\rightarrow$ $$ 解释器 $$ $\rightarrow$ $$ 操作系统。像“同声传译”，边运行边解释，跨平台性强。
* 编译型 (Golang/C++): 源代码 $$ $\rightarrow$ $$ 编译器 $$ $\rightarrow$ $$ 可执行文件 $$ $\rightarrow$ $$ 操作系统。像“翻译完一整本书再出版”，运行速度极快。先整体翻译成机器码再运行。像“翻译完整本书再出版”，运行效率极高。

#### 2. 环境选择：本地 vs 云端

* PyCharm (专业本地 IDE): \* 强项： 智能补全、强大的调试器（Debugger）、集成的版本控制界面。
  * 场景： 适合中大型项目、需要深度开发和管理复杂逻辑的场景。
* 云端环境 (Google Colab): 无需配置，内置数据科学库。适合实验、分享代码和快速原型。

***

### 二、 核心实战：搭建与运行 (10 分钟)

#### 1. PyCharm 项目初始化

1. 新建项目 (New Project): 打开 PyCharm，选择项目路径。
2. 虚拟环境 (Venv): 重点讲解为什么要用虚拟环境（隔离不同项目的依赖，保持系统洁净）。
3. 配置 Interpreter: 关联已安装的 Python 解释器。

#### 2. 运行首个程序

* 右键点击项目目录 $$ $\rightarrow$ $$ `New` $$ $\rightarrow$ $$ `Python File` $$ $\rightarrow$ $$ 命名为 `main.py`。
* 编写代码：`print("Hello PyCharm")`。
* 运行技巧： 点击右上角绿色三角形或使用快捷键 `Shift + F10`。

***

### 三、 Git 与 GitHub：代码的“后悔药”与协作 (15 分钟)

#### 1. 版本控制的核心概念

* Repository (仓库): 存放项目的文件夹。
* Commit (提交): 代码的时间戳/快照。
* Branch (分支): 独立的工作空间。

#### 2. Git 基础命令流

| **步骤** | **命令**                       | **描述**              |
| ------ | ---------------------------- | ------------------- |
| 初始化    | `git init`                   | 将当前目录变为 Git 仓库      |
| 暂存     | `git add .`                  | 准备提交当前所有更改          |
| 保存     | `git commit -m "feat: init"` | 永久记录当前状态            |
| 同步     | `git push`                   | 将本地代码推送到云端 (GitHub) |

#### 3. 协同开发流 (PR 模式)

1. Fork: 把别人的代码仓库“克隆”一份到自己名下。
2. Clone: 在 PyCharm 中点击 `Get from VCS`，粘贴 URL 下载到本地。
3. Pull Request (PR): 向主项目提交合并请求。

### 四、 总结与课后任务 (5 分钟)

#### 核心知识回顾

* Python 是解释型语言，依赖解释器运行。
